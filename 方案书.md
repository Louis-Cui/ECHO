# 数字人情感陪伴系统 — 方案书

## 一、项目定位

基于 Human-Live2D 底座 + 自研情感大脑的数字人陪伴系统。不是从零造数字人，而是给数字人注入"情感中枢"。

### 核心差异点

```
传统数字人：
  语音识别 → LLM 回复 → TTS 合成 → Live2D 对口型
  ↑ 无情绪感知，无记忆，无主动交互

本系统：
  多模态输入（语音/文本/视频）
    → 情绪融合（文本情绪 + 语音情绪 + 面部情绪）
    → 带记忆上下文的 LangGraph Agent
    → 情绪感知 TTS（音色/语速随情绪变化）
    → Live2D 驱动口型+表情
    → RAG 自学习闭环
    → 主动关怀调度器（非空闲状态触发）
```

## 三、核心工作流（树状展开）

### 3.1 实时对话流程

```
用户说话
├── 文本输入
│   └── [POST /chat]
│
├── 语音输入
│   ├── [POST /asr/upload]
│   │   ├── SenseVoiceSmall 推理
│   │   │   ├── ASR 输出文字
│   │   │   └── SER 输出情绪标签 + 置信度
│   │   └── → 语音EmotionOutput
│   └── → [POST /chat] 带情绪输入
│
└── 视频输入（可选）
    ├── [POST /emotion/face]
    │   ├── FER 面部情绪检测
    │   │   ├── 载入图像
    │   │   ├── 检测人脸（MTCNN）
    │   │   └── 输出 7 种情绪 + 置信度
    │   └── → 面部EmotionOutput
    └── → [POST /chat] 带情绪输入

[POST /chat] 处理链
├── 1. 情绪融合 (EmotionFusion.fuse)
│   ├── 输入：文本情绪 / 语音情绪 / 面部情绪（可选）
│   ├── VETO 规则检查
│   │   ├── 规则1: 文本负面 > 0.7 → 覆写其他模态
│   │   └── 规则2: 文本含爱意关键词 → 覆写为 love
│   ├── 加权投票
│   │   ├── 文本权重 0.4 × 置信度
│   │   ├── 语音权重 0.35 × 置信度
│   │   └── 面部权重 0.25 × 置信度
│   └── → 最终情绪标签 + 来源明细
│
├── 2. LangGraph Agent 工作流
│   ├── Node: analyze_emotion
│   │   ├── LLM 零样本情绪分类
│   │   ├── 与外部情绪融合结果交叉验证
│   │   └── → 结构化情绪标签
│   │
│   ├── Node: retrieve_memory
│   │   ├── Chroma 向量检索
│   │   │   ├── 按用户ID 过滤
│   │   │   ├── 语义相似度排序
│   │   │   ├── 权重衰减（近期高权 → 优先）
│   │   │   └── → Top-5 记忆片段
│   │   └── 短期上下文（Redis/内存）
│   │       └── → 最近 N 条对话
│   │
│   ├── Node: build_prompt
│   │   ├── 系统提示（"沐光"人格）
│   │   ├── 记忆上下文拼接
│   │   ├── 当前情绪标签 + 风格调整
│   │   └── → 完整 Prompt
│   │
│   ├── Node: generate_response
│   │   ├── DeepSeek API 调用
│   │   ├── 流式/非流式输出
│   │   └── → 回复文本
│   │
│   ├── Node: determine_response_emotion
│   │   ├── 分析回复文本情绪
│   │   ├── → 回复情绪标签
│   │   └── 传递给 TTS
│   │
│   └── Node: store_memory
│       ├── (用户输入, 情绪, 回复) → Chroma 入库
│       ├── 更新用户情绪状态（供主动关怀用）
│       └── → 记忆持久化
│
└── 3. TTS 合成 (CosyVoice)
    ├── 根据回复情绪选择音色参数
    │   ├── happy → speed 1.1, pitch 1.2
    │   ├── sad → speed 0.85, pitch 0.8
    │   ├── love → speed 0.9, pitch 1.1
    │   └── ... 9 种情绪映射
    ├── CosyVoice 推理
    ├── → WAV 音频 Base64
    └── 返回前端 → Live2D 播放 + 口型同步
```

### 3.2 主动关怀流程

```
主动关怀调度器 (ActiveCareScheduler)
├── 每 5 分钟检查一次 (APScheduler)
│
├── 遍历所有已注册用户
│   │
│   ├── 规则1: 长时间未活跃
│   │   ├── 条件：last_active > 8 小时
│   │   ├── 状态：触发问候
│   │   └── 动作：Agent 生成早安/晚安问候
│   │
│   ├── 规则2: 情绪持续低落
│   │   ├── 条件：连续 3 次负面情绪
│   │   ├── 阈值：负面置信度 ≥ 0.6
│   │   └── 动作：Agent 生成安慰/关心话术
│   │
│   └── 规则3: 常规关心
│       ├── 条件：last_active > 4 小时，无低情绪
│       ├── 状态：触发日常问候
│       └── 动作：Agent 生成问候消息
│
└── 触发后更新用户状态
    └── 下次检查跳过冷却期
```

### 3.3 RAG 自学习闭环

```
对话完成
├── 存储
│   ├── Chroma 向量库
│   │   ├── embedding: 用户输入 + 回复
│   │   ├── metadata: user_id, emotion, timestamp, weight=1.0
│   │   └── 持久化到磁盘
│   └── SQLite (日志归档)
│
├── 反馈收集
│   ├── 用户点赞/点踩 (Human-Live2D 支持)
│   ├── 点赞 → 该记忆 weight += 0.1
│   └── 点踩 → 该记忆 weight -= 0.1
│
└── 下次检索
    ├── 高权重记忆优先召回
    ├── 近期记忆有额外加分
    └── 负面情绪记忆在同类情绪场景下权重更高
```

## 八、答辩亮点包装

```
核心技术爆点（按含金量排序）：

1. 多模态情绪融合 —— VETO 规则 + 加权投票
   技术创新点：不是简单平均，而是优先级裁决

2. RAG 自学习闭环 —— 用户反馈调权
   技术创新点：记忆不是死的，权重随反馈动态变化

3. 情绪感知 TTS —— 9 种情绪 → 不同音色/语速
   技术创新点：不只是"说"，是"怎么说的"

4. 主动关怀调度器 —— 不用等用户开口
   技术创新点：从"被动应答"到"主动陪伴"

5. LangGraph Agent 工作流 —— 可观测/可编排
   技术创新点：每步可 debug，可替换节点
```
